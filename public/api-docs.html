Getting Started

Welcome to the TabulariumCMS API. This is a RESTful API that returns JSON responses and follows standard HTTP conventions for methods, headers, and status codes.

Base URL

https://your-domain.com/api

Response Format

All API responses are returned in JSON format. Successful responses will include the requested data along with appropriate status codes. Error responses will include an error message and details.

Note: Some endpoints are public and don't require authentication, while others require a valid API key. Check each endpoint's documentation for specific requirements.

Authentication

Protected API endpoints require authentication using an API key. You can generate API keys from the admin panel.

Using API Keys

Include your API key in the request header:

X-API-Key: your-api-key-here

Permissions

Each API key has specific permissions that define what resources it can access and what actions it can perform. Actions include:

  • read - View resources
  • create - Create new resources
  • update - Modify existing resources
  • delete - Remove resources
Public Endpoints: Some endpoints like product listings, search, and cart operations don't require API keys and can be accessed without authentication.

Products

The Products API lets you browse and filter the product catalog. These endpoints are public and don't require authentication.

GET /api/products

Get a paginated list of products with optional filtering and sorting.

Query Parameters

Parameter Type Description
page integer Page number (default: 1)
limit integer Items per page (default: 12, max: 50)
price_min float Minimum price filter
price_max float Maximum price filter
categories array Filter by category IDs
type string Product type (physical, virtual, bundle, room, timeslot, ticket)
in_stock boolean Show only in-stock products (true/false)
sort string Sort order: newest, price_asc, price_desc, name_asc

Example Request

GET /api/products?page=1&limit=12&price_min=10&price_max=100&sort=price_asc

Example Response

{ "success": true, "data": [ { "id": 1, "name": "Premium Widget", "slug": "premium-widget", "shortDescription": "A high-quality widget for your needs", "grossPrice": 49.99, "stock": 100, "isFeatured": true, "type": "physical", "isRequestOnly": false, "requestEmail": null, "mainImage": { "url": "/uploads/media/widget.jpg", "alt": "Premium Widget" }, "category": { "id": 5, "name": "Widgets", "slug": "widgets" } } ], "pagination": { "page": 1, "limit": 12, "total": 45, "totalPages": 4, "hasMore": true } }
GET /api/products/categories

Get all active product categories.

Example Response

[ { "id": 1, "name": "Electronics", "slug": "electronics" }, { "id": 2, "name": "Clothing", "slug": "clothing" } ]
GET /api/products/types

Get available product types.

Example Response

[ { "value": "physical", "label": "Physical Product" }, { "value": "virtual", "label": "Digital Product" }, { "value": "bundle", "label": "Bundle" }, { "value": "room", "label": "Room Booking" }, { "value": "timeslot", "label": "Time Slot" }, { "value": "ticket", "label": "Ticket" } ]

Articles

The Articles API provides access to product and content articles. Public endpoints don't require authentication.

GET /api/articles/public

Get public articles with filtering and pagination. Similar to products endpoint but with more detailed filtering options.

Query Parameters

Parameter Type Description
search string Search in name, SKU, or description
category integer Filter by category ID
type string Filter by product type
price_min float Minimum price
price_max float Maximum price
in_stock boolean Only show items in stock
sort string newest, price_asc, price_desc, name_asc, featured
page integer Page number (default: 1)
limit integer Results per page (default: 12, max: 100)
GET /api/articles

Get all articles. Requires authentication and read permission.

Authentication Required: This endpoint requires a valid API key with read permission for Articles.
GET /api/articles/{id}

Get a specific article by ID. Requires authentication.

URL Parameters

Parameter Type Description
id integer Article ID
POST /api/articles

Create a new article. Requires authentication and create permission.

Request Body

{ "title": "New Article", "sku": "ART-001", "price": "29.99", "content": "Article description", "isActive": true }
PUT PATCH /api/articles/{id}

Update an existing article. Requires authentication and update permission.

DELETE /api/articles/{id}

Delete an article. Requires authentication and delete permission.

GET /api/articles/{id}/stock

Get stock information for a specific article. Requires authentication.

Shopping Cart

Manage shopping cart operations. These endpoints are public but use session-based cart tracking.

GET /api/cart

Get the current cart for the session.

Example Response

{ "items": [ { "id": 1, "article": { "id": 10, "name": "Premium Widget", "slug": "premium-widget", "sku": "WID-001", "image": "/uploads/media/widget.jpg" }, "quantity": 2, "price": 49.99, "subtotal": 99.98 } ], "totals": { "subtotal": 99.98, "itemCount": 2 } }
POST /api/cart/add

Add a product to the cart.

Request Body

Parameter Type Required Description
article_id integer Yes ID of the product to add
quantity integer No Quantity to add (default: 1, min: 1)

Example Request

{ "article_id": 10, "quantity": 2 }

Example Response

{ "success": true, "message": "Product added to cart", "cart": { "items": [...], "totals": { "subtotal": 99.98, "itemCount": 2 } } }
PUT /api/cart/update/{itemId}

Update the quantity of a cart item.

Request Body

{ "quantity": 3 }
DELETE /api/cart/remove/{itemId}

Remove an item from the cart.

URL Parameters

Parameter Type Description
itemId integer Cart item ID to remove

Checkout

Process orders and handle payments.

GET /api/checkout/methods

Get available payment and shipping methods.

Example Response

{ "payment_methods": [ { "id": 1, "name": "Credit Card", "type": "stripe", "description": "Pay with credit or debit card", "fee": 0 }, { "id": 2, "name": "PayPal", "type": "paypal", "description": "Pay with PayPal", "fee": 0 } ], "shipping_methods": [ { "id": 1, "name": "Standard Shipping", "description": "5-7 business days", "price": 9.99, "delivery_time": "5-7 days" }, { "id": 2, "name": "Express Shipping", "description": "2-3 business days", "price": 19.99, "delivery_time": "2-3 days" } ] }
POST /api/checkout/process

Process the checkout and create an order.

Request Body

Parameter Type Required Description
payment_method_id integer Yes Selected payment method ID
shipping_method_id integer Yes Selected shipping method ID
email string Yes Customer email
name string Yes Customer name
shipping_address string Yes Shipping address
billing_address string No Billing address (defaults to shipping address)
payment_data object No Payment-specific data (tokens, etc.)

Example Request

{ "payment_method_id": 1, "shipping_method_id": 1, "email": "customer@example.com", "name": "John Doe", "shipping_address": "123 Main St, City, State 12345", "billing_address": "123 Main St, City, State 12345", "payment_data": {} }

Example Response (Success)

{ "success": true, "message": "Payment processed successfully", "order_id": 42, "transaction_id": "txn_abc123", "instructions": null }

Example Response (Error)

{ "success": false, "error": "Payment was declined" }

Search

Search across products, articles, and other content. These endpoints include rate limiting (10 requests per minute per IP).

GET /api/search

Search for products and content.

Query Parameters

Parameter Type Required Description
q string Yes Search query (2-100 characters)
limit integer No Max results (default: 5, max: 20)

Example Request

GET /api/search?q=widget&limit=10

Example Response

{ "success": true, "query": "widget", "data": { "products": [...], "articles": [...], "total": 15 }, "total": 15 }
Rate Limiting: This endpoint is limited to 10 requests per minute per IP address. You'll receive a 429 status code if you exceed this limit.
GET /api/search/suggestions

Get autocomplete suggestions for search queries.

Query Parameters

Parameter Type Required Description
q string Yes Partial search query (2-100 characters)
limit integer No Max suggestions (default: 10, max: 20)

Example Response

{ "success": true, "query": "wid", "data": [ "widget", "widgets premium", "wide screen" ] }

Error Handling

The API uses standard HTTP status codes to indicate success or failure of requests.

HTTP Status Codes

Code Status Description
200 OK Request succeeded
201 Created Resource created successfully
400 Bad Request Invalid request parameters or data
401 Unauthorized Missing or invalid API key
403 Forbidden API key lacks required permissions
404 Not Found Resource doesn't exist
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Something went wrong on the server

Error Response Format

When an error occurs, the API returns a JSON response with an error message:

{ "error": "Description of what went wrong", "success": false }

Common Error Scenarios

Missing API Key

{ "error": "No API key found" }

Insufficient Permissions

{ "error": "Permission denied: create on Article" }

Resource Not Found

{ "error": "Article not found" }

Validation Error

{ "errors": { "title": "This field is required", "price": "Price must be a valid number" } }

Rate Limit Exceeded

{ "success": false, "error": "Rate limit exceeded. Please try again in a minute.", "data": [] }
Best Practice: Always check the HTTP status code and handle errors appropriately in your application. Include error handling for network failures, timeouts, and unexpected responses.

Additional Information

Pagination

Most list endpoints support pagination using these parameters:

  • page - The page number (starts at 1)
  • limit - Number of items per page

Paginated responses include a pagination object with details:

{ "data": [...], "pagination": { "page": 1, "limit": 20, "total": 100, "pages": 5 } }

Rate Limiting

Search endpoints are rate-limited to 10 requests per minute per IP address. When you hit the rate limit, you'll receive a 429 status code. Wait a minute before trying again.

CORS

The API supports CORS (Cross-Origin Resource Sharing) for browser-based applications. Make sure your domain is configured in the system if you're accessing the API from a web application.

Data Formats

  • Dates: All dates are in ISO 8601 format (YYYY-MM-DD HH:MM:SS)
  • Prices: Decimal values as strings or floats
  • Booleans: true/false or 1/0 for query parameters

Best Practices

  • Store your API key securely and never expose it in client-side code
  • Use pagination to limit response sizes
  • Implement proper error handling for all API calls
  • Cache responses when appropriate to reduce API calls
  • Respect rate limits to ensure service availability

Need Help?

If you run into issues or have questions about the API:

  • Check the error message for specific details about what went wrong
  • Verify your API key has the correct permissions
  • Review the request parameters and format
  • Contact support if you need additional assistance
